# 轻小说校对排版检查清单（agent3执行）

每次完成创作后，校对必须严格按照以下 checklist 逐一检查，全部检查通过才能推送到GitHub：

---

## 1. 章节结构检查（必须过）
- [ ] 确认章节编号连续：从第一章开始，1、2、3、4...不准跳号、不准重复编号
- [ ] 确认每个章节都有明确的二级标题（## 第一章 XXX）
- [ ] 确认每个章节字数大体均衡：尽量控制在每章1.2w-1.8w之间，不准某一章只有两三千字另一章两三万字
- [ ] 确认标题格式统一：全部用 `## 第X章 标题文字` 格式，不准混用不同层级标题

## 2. 内容完整性检查
- [ ] 确认故事从头到尾完整：开头、发展、高潮、结局齐全，不准写一半没结尾
- [ ] 确认没有缺页漏段：复制粘贴的时候不准漏内容，从第一章标题到最后结尾一字不少
- [ ] 确认作者要求都满足：总字数够不够（现在要求至少10万字）、是不是避开了烂大街套路、文笔有没有达标

## 3. 格式排版检查
- [ ] 确认标题和正文之间空一行，每个章节之间空两行，排版清晰
- [ ] 确认段落之间空行，不准一大段从头到尾不分段
- [ ] 检查错别字和标点符号：中文用全角标点，引号、括号配对正确，明显错别字要修正
- [ ] 检查语句通顺：不通顺的句子帮忙改顺，不改变原意前提下优化流畅度

## 4. 文件名和推送检查
- [ ] 确认文件名正确：用中文书名+`.md`后缀，不准乱起名不准错字
- [ ] 确认文件保存在正确路径（当前工作区根目录）
- [ ] 推送前检查git status，确认文件已经add，commit信息清晰明了
- [ ] 推送后可以简单pull一下验证，确认GitHub上已经更新成功，本地和远程一致

## 5. 收尾工作
- [ ] 校对完成推送成功后，必须清理所有子agent进程，释放资源，不准残留占用
- [ ] 完成后发通知给用户，说清楚书名、总字数、章节数，确认推送成功

---

**要求：每次校对必须逐一打勾确认，不准跳检查，错一个都不能推送！**
